# Examples

Fully-worked sample `.claude/` setups for each stack profile. Use as
reference when building your own.

## Layout

```
examples/
  java-spring-orchestrator/    ← Spring Boot REST + JPA + Kafka
  python-fastapi-todo/         ← FastAPI + SQLAlchemy + Alembic
  ts-next-blog/                ← Next.js 15 + Drizzle + Vitest
  rust-axum-api/               ← Axum + sqlx + Postgres
  go-gin-api/                  ← Gin + sqlc + Postgres
  monorepo-turbo/              ← Turborepo with 3 apps + 2 packages
  data-science-dbt/            ← Python + dbt + Jupyter
```

## How examples are built

Each example is a real (small) project. Its `.claude/` was generated by:

```bash
cd examples/<name>
conjure init existing --profile=<stack> .
# then hand-tuned to fit the example's needs
```

## How to use as reference

1. Pick the example closest to your stack.
2. Read its `CLAUDE.md` — see how non-negotiables, build commands, routing
   come together for that stack.
3. Look at the skill bodies — they're filled in (not template scaffolds).
4. Copy patterns into your own project's `.claude/`.

## What examples are NOT

- They are not "starter templates" — don't copy the whole directory.
- They are not maintained for runtime use — they're config showcases.
- They may lag behind the kit's latest skills.

## Adding a new example

See `CONTRIBUTING.md` — every new profile should ship with an example.
