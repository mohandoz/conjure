# Conjure Skill-Firing Telemetry

Conjure ships optional, local-only, PII-free skill telemetry. When opted in, every skill
invocation is appended as a single JSON line to `{project}/.claude/telemetry/skill-events.jsonl`
in the project that uses the generated harness. No data leaves your machine — there is no
server, no HTTP call, and no aggregation outside your own filesystem. The log is user-owned,
human-readable, and gitignored by default.

---

## Opt-In

Telemetry is **off by default**. To enable it, set `CONJURE_TELEMETRY=1` in your project's
`.claude/settings.json` `env` block:

```json
{
  "env": {
    "CONJURE_TELEMETRY": "1"
  }
}
```

Once set, every skill invocation (both tool-interface and slash-command) appends one JSONL
record to `.claude/telemetry/skill-events.jsonl`. No restart is needed — the hook checks the
env var on every invocation.

---

## DO_NOT_TRACK

If `DO_NOT_TRACK=1` is set in your environment, the hook exits immediately without writing
anything. This check runs **before** the `CONJURE_TELEMETRY` check, per the Unix
[DO_NOT_TRACK convention](https://consoledonottrack.com/). The hook exits 0 silently — Claude
continues normally with no indication that telemetry was suppressed.

---

## Event Schema

Each line in `skill-events.jsonl` is a JSON object with exactly five fields:

| Field | Type | Description |
|-------|------|-------------|
| `ts` | ISO 8601 UTC string | Timestamp of the skill invocation (e.g. `"2026-05-25T03:00:00.000Z"`) |
| `session_id` | string | Claude Code session identifier — passed by CC, not generated by the hook |
| `event` | `"skill_invoke"` or `"skill_typed"` | `skill_invoke` = PreToolUse/Skill tool call; `skill_typed` = UserPromptExpansion slash command |
| `skill` | string | Skill name only — **never** includes arguments (arguments may contain sensitive data) |
| `project_cwd` | string | Working directory at time of invocation, used as project identifier |

**Example line:**

```json
{"ts":"2026-05-25T03:00:00.000Z","session_id":"abc123","event":"skill_invoke","skill":"gsd-execute-phase","project_cwd":"/Users/x/myproject"}
```

---

## Log Location

```
{project}/.claude/telemetry/skill-events.jsonl
```

- Created on first write — no pre-creation needed.
- Append-only (`>>`): each invocation adds one line; existing lines are never modified.
- User-owned: the file lives inside your project directory with your normal OS permissions.
- Gitignored: `templates/.gitignore.tmpl` includes `.claude/telemetry/` so generated projects
  exclude this file from version control automatically (see [Gitignore](#gitignore) below).

---

## No-Egress Guarantee

The hook uses **no HTTP, fetch, curl, sockets, or any network call** — it is purely a local
filesystem append using Node.js `appendFileSync`. Only `node:fs` and `node:path` are imported.

You can verify this yourself at any time by running:

```sh
grep -E 'curl|fetch|http|socket|XMLHttpRequest' templates/hooks-nodejs/skill-telemetry.mjs
```

This command returns **no output** (exit code 1) when the hook is clean. If any network pattern
appears, the grep will print it — and the CI no-egress test (`TLMY-03`) will fail the build.

## Retire-List

`conjure audit --retire-list` reads `skill-events.jsonl` from the target project and surfaces
skills with 0 loads across the last 30 days:

```
── Skill Retire-List ──────────────────────────────────
  Skill                               Loads   Status
  -----                               -----   ------
  gsd-execute-phase                      42   [active]
  gsd-quick                               7   [active]
  old-refactor-helper                     0   [retire?]
```

Skills marked `[retire?]` are candidates for removal from your `.claude/skills/` directory.
If the log file is absent, the command prints an advisory and skips the section.

---

## Gitignore

`templates/.gitignore.tmpl` includes `.claude/telemetry/` in the Conjure-managed runtime
state block. Projects generated by `conjure init` automatically exclude this directory from
git. If you added Conjure to an existing project, add the following to your `.gitignore`:

```gitignore
.claude/telemetry/
```
