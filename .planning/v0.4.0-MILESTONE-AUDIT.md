## Tech Debt Register

### Phase 15: Release Pipeline
1. **Docker + Homebrew sequential coupling** — Docker push failure silently skips Homebrew bump. Fix: split into separate jobs with `needs:` and `if: always()`. Deferred to v0.4.x.
2. **No HOMEBREW_TAP_GITHUB_TOKEN preflight** — Missing secret causes cryptic failure. Fix: add `gh secret list | grep HOMEBREW_TAP_GITHUB_TOKEN` preflight step. Deferred to v0.4.x.

### Phase 14: Docker + Windows CI
3. **shellcheck absent on windows-latest** — Gracefully handled; tests silently skip. Acceptable for v0.4.0.

### Phase 11: Skill Publishing
4. **SKILL-04 env forwarding** — `--to <org/repo>` uses `TARGET_REPO` env var rather than positional arg to worker script. Functional but implicit. Deferred to v0.5.0.

### Phase 13: Homebrew Tap
5. **Formula sha256 PLACEHOLDER** — Expected pre-release state. No issue after first `bump-homebrew-formula-action` fires. Not a debt item for this audit.

## Pre-Release Checklist (human actions before v0.4.0 tag)

1. Create `mohandoz/homebrew-conjure` tap repo with initial `Formula/conjure.rb`
2. Set `HOMEBREW_TAP_GITHUB_TOKEN` in repo secrets (PAT with tap repo write access)
3. Update `VERSION` file to `0.4.0`
4. Update `marketplace.json` and `plugin.json` version fields to `0.4.0`
5. Update `Formula/conjure.rb` url to point to `v0.4.0` tarball
6. Add `CHANGELOG.md` entry for `[0.4.0]`
7. Run `conjure publish` to update marketplace.json SHA
8. Push `v0.4.0` tag → release.yml fires → verify all artifacts
