  <name>Task 1: Create Dockerfile</name>
  <files>Dockerfile</files>
  <action>
    Create Dockerfile at the repo root. Follow the exact structure from RESEARCH.md
    "Full Dockerfile Structure" section. Every decision here implements D-01 through
    D-03 from CONTEXT.md.

    Layer order (must be this order to avoid pitfalls from RESEARCH.md):

    1. FROM debian:bookworm-slim

    2. Single RUN layer for all apt deps (D-01, D-02). Combine update + install +
       NodeSource setup + nodejs install + cleanup into one RUN to keep layer size
       minimal. Include --no-install-recommends on every apt-get install call.
       Packages: ca-certificates curl gnupg jq shellcheck. Then run the NodeSource
       setup_22.x script, then install nodejs. Clean up with apt-get clean and
       rm -rf /var/lib/apt/lists/*. Do NOT split into multiple RUN commands — the
       NodeSource pitfall (RESEARCH.md Pitfall 2) requires setup script + nodejs
       install in one layer.

    3. RUN useradd -m -u 1000 -s /bin/bash conjure  (D-03, DOCK-02)

    4. ENV CONJURE_HOME=/usr/local/share/conjure

    5. COPY directives — copy each source directory individually into $CONJURE_HOME:
       COPY cli/       $CONJURE_HOME/cli/
       COPY scripts/   $CONJURE_HOME/scripts/
       COPY profiles/  $CONJURE_HOME/profiles/
       COPY compliance/ $CONJURE_HOME/compliance/
       COPY migrations/ $CONJURE_HOME/migrations/
       COPY templates/ $CONJURE_HOME/templates/
       COPY lib/       $CONJURE_HOME/lib/
       COPY VERSION    $CONJURE_HOME/VERSION

    6. RUN block that:
       a. chmod +x $CONJURE_HOME/cli/conjure
       b. Uses printf to write the wrapper at /usr/local/bin/conjure. The wrapper
          must mirror Formula/conjure.rb exactly:
            #!/bin/bash
            export CONJURE_HOME=/usr/local/share/conjure
            exec /usr/local/share/conjure/cli/conjure "$@"
          Use printf with literal paths (not shell variables that won't expand in
          RUN context correctly). Then chmod +x /usr/local/bin/conjure.
       c. mkdir -p /work  (as root, before USER switch — RESEARCH.md Pitfall 3)

    7. USER conjure

    8. WORKDIR /work

    9. ENTRYPOINT ["conjure"]

    Add inline comments on the non-root user step referencing DOCK-02 and D-03, and
    on the NodeSource step referencing the single-layer requirement.

    Do NOT add a CMD instruction — ENTRYPOINT ["conjure"] is sufficient; the user
    passes the subcommand as the Docker run argument (e.g., `docker run conjure:local version`).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build context → Docker image | Files COPYed from repo root; .dockerignore prevents secrets or planning artifacts entering the image |
| caller → container | User passes --user $(id-u):$(id-g) at runtime; container cannot escalate beyond conjure UID 1000 by default |
| NodeSource setup_22.x script | curl pipe to bash at build time; not a user-facing installer (CLAUDE.md constraint applies only to Conjure's own scripts) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Elevation of Privilege | USER conjure (UID 1000) | mitigate | Non-root user enforced in Dockerfile per DOCK-02; runtime --user flag for volume ownership |
| T-14-02 | Tampering | CONJURE_HOME env var | accept | ENV is baked in image; only overridable via --env at runtime (requires Docker access already) |
| T-14-03 | Information Disclosure | Build context | mitigate | .dockerignore excludes .planning/, .git/, .github/ preventing secrets and planning docs entering image |
| T-14-SC | Tampering | NodeSource curl pipe | accept | Build-time only; NodeSource is a well-known distribution channel; not a user-facing installer pattern |
</threat_model>
