    Create .github/workflows/docker.yml. This workflow has a single job `build` on
    ubuntu-latest with the following step order (exact step names matter for CI log
    readability):

    Trigger: workflow_dispatch only (D-04). No push:, no pull_request:. Phase 14
    does not wire Docker into automated CI — that is Phase 15's job.

    Steps in order:

    1. actions/checkout@v4

    2. "Set up Docker daemon (containerd snapshotter)"
       uses: docker/setup-docker-action@v4
       with daemon-config JSON enabling containerd-snapshotter: true
       This step is REQUIRED before buildx — without it, loading a multi-platform
       image locally silently drops the arm64 layer (RESEARCH.md Pitfall 1).

    3. "Set up QEMU"
       uses: docker/setup-qemu-action@v3
       (Required for linux/arm64 cross-compilation on the amd64 runner)

    4. "Set up Docker Buildx"
       uses: docker/setup-buildx-action@v3

    5. "Build multi-arch image (no push)"
       uses: docker/build-push-action@v6
       with:
         context: .
         platforms: linux/amd64,linux/arm64
         load: true
         tags: conjure:local
       No push: true, no registry credentials. Phase 14 builds locally only.

    6. "Smoke test — conjure version"
       run: docker run --rm conjure:local version

    7. "Scaffold fixture for audit"
       run: |
         mkdir -p /tmp/fixture
         cp -r tests/fixtures/python-fastapi/. /tmp/fixture/
       (Uses python-fastapi fixture — confirmed present in repo at tests/fixtures/python-fastapi/)

    8. "Smoke test — conjure audit"
       run: |
         docker run --rm \
           -v /tmp/fixture:/work \
           --user "$(id -u):$(id -g)" \
           conjure:local audit /work
       The --user flag passes caller ownership through to volume writes (D-06).
       The audit command runs read-only against the fixture so no ownership issue
       arises even if the runner UID differs from conjure (UID 1000).

    9. "Assert image size ≤200 MB"
       run: |
         SIZE=$(docker image inspect --format '{{.Size}}' conjure:local)
         LIMIT=$((200 * 1024 * 1024))
         echo "Image size: $SIZE bytes (limit: $LIMIT)"
         [ "$SIZE" -lt "$LIMIT" ] || { echo "FAIL: image exceeds 200 MB"; exit 1; }
       Uses .Size which is uncompressed bytes. LIMIT is 209,715,200 bytes
       (RESEARCH.md Pitfall 4 — interpret 200 MB as uncompressed on-disk size).

    Action versions to use (from RESEARCH.md "Action Versions" table):
    - docker/setup-docker-action@v4
    - docker/setup-qemu-action@v3
    - docker/setup-buildx-action@v3
    - docker/build-push-action@v6
    - actions/checkout@v4

    Add a comment at the top of the file noting that SHA pinning is Phase 15 (WR-01).
    Do NOT add any registry login step, push: true, or ghcr.io references.
