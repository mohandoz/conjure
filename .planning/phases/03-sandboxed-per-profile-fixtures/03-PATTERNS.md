## Pattern Assignments

### `tests/lib/sandbox.sh` (utility, sourced library)

**Analog:** `lib/mutate.sh`

The only sourced bash library in the project. Establishes the contract: no shebang line (it is sourced, not executed), POSIX bash 3.2+ compatible, exposes named functions, avoids global side effects on source.

**Header / sourcing contract pattern** (`lib/mutate.sh` lines 1–13):
```bash
#!/usr/bin/env bash
# lib/mutate.sh — sourced mutation chokepoint for Conjure.
# Source this file; call mutate_mkdir, mutate_cp, mutate_write, mutate_summary.
# Requires: DRY_RUN env var (0=live, 1=dry); set -u safe via ${DRY_RUN:-0}.
# POSIX bash 3.2+ compatible. No associative arrays, no mapfile, no local -n.
#
# Usage from any script:
#   source "$CONJURE_HOME/lib/mutate.sh"
#   mutate_mkdir  <dir>
```

**Function definition pattern** (`lib/mutate.sh` lines 21–28):
```bash
mutate_mkdir() {
  if [ "${DRY_RUN:-0}" = "1" ]; then
    echo "[dry-run] would mkdir $1"
    CONJURE_DRY_MUTATION_COUNT=$((CONJURE_DRY_MUTATION_COUNT + 1))
    return 0
  fi
  mkdir -p "$1"
}
```

**Guard-against-missing-var pattern** (`lib/mutate.sh` line 16):
```bash
CONJURE_DRY_MUTATION_COUNT="${CONJURE_DRY_MUTATION_COUNT:-0}"
```

**What sandbox.sh must copy from this analog:**
- No shebang line — it is `source`d, not executed directly.
- `local` for all function-local vars (POSIX-compatible scoping).
- Expose `SANDBOX_DIR` as a global output variable (mirrors `CONJURE_DRY_MUTATION_COUNT` global counter pattern).
- Usage comment block at top listing the public interface.

**mktemp + trap pattern from the direct codebase reference** (`tests/run.sh` lines 187–188):
```bash
TMPDIR_TARGET="$(mktemp -d)"
trap 'rm -rf "$TMPDIR_TARGET"' EXIT
```

**sandbox_setup implementation to produce** (from RESEARCH.md Pattern 1):
```bash
# tests/lib/sandbox.sh — sourced library (not executable)
# Usage: source "$CONJURE_HOME/tests/lib/sandbox.sh"
#        sandbox_setup <fixture_dir>
# Sets SANDBOX_DIR. Exports HOME/XDG_CONFIG_HOME/CLAUDE_CONFIG_DIR/PATH.
# Callers must: trap 'rm -rf "$SANDBOX_DIR"' EXIT
# CONJURE_HOME must remain set to the real kit location — do NOT override it.

sandbox_setup() {
  local fixture_dir="$1"
  SANDBOX_DIR="$(mktemp -d)"
  cp -r "$fixture_dir/." "$SANDBOX_DIR/"
  export HOME="$SANDBOX_DIR"
  export XDG_CONFIG_HOME="$SANDBOX_DIR"
  export CLAUDE_CONFIG_DIR="$SANDBOX_DIR"
  export PATH="$CONJURE_HOME/cli:/usr/local/bin:/usr/bin:/bin"
}
```

### `tests/fixtures/_broken/CLAUDE.md` (broken fixture, file-I/O)

**Analog:** None — deliberately invalid content.

**Key constraint:** Must be 201+ lines to trigger the `err` path (exit 2) in `audit-setup.sh`, not just the `warn` path (exit 1). The `audit-setup.sh` threshold is:
- `≤100 lines` → `ok` (exit 0)
- `101–200 lines` → `warn` (exit 1)
- `>200 lines` → `err "HARD CAP exceeded — trim"` (exit 2)

From `scripts/audit-setup.sh` lines 26–29:
```bash
if [ "$LINES" -le 100 ]; then ok "CLAUDE.md: $LINES lines (≤100)"
elif [ "$LINES" -le 200 ]; then warn "CLAUDE.md: $LINES lines (within hard cap but over practical limit)"
else err "CLAUDE.md: $LINES lines (HARD CAP exceeded — trim)"
fi
```

The `_broken/CLAUDE.md` must have a valid `.claude/` directory (so audit proceeds past the early-exit guard at `audit-setup.sh` line 44) but a CLAUDE.md padded to 201+ lines. The simplest approach: copy a green fixture's `.claude/` directory, pad the CLAUDE.md with blank filler lines.

## Critical Implementation Notes for Planner

1. **Fixture directories are generated, not hand-written.** The plan must include a step that runs `scripts/regen-fixtures.sh` (or equivalent per-profile `conjure init` invocations) and commits the output. The 9 fixture directories are artifacts, not manual source files.

2. **regen-fixtures.sh must write a seed CLAUDE.md before calling `conjure init`.** Without this, `profiles/*/apply.sh` silently skips fragment append and audit exits with `CLAUDE.md missing` error. (RESEARCH.md Critical Finding 1, Pitfall 1)

3. **`_broken/CLAUDE.md` must be 201+ lines, not 105+ lines.** The 100-line "practical limit" triggers WARN (exit 1); only >200 lines triggers ERR (exit 2). EXPECT pattern `HARD CAP exceeded` matches only the ERR path. (RESEARCH.md Critical Finding 2, Pitfall 2)

4. **`sandbox.sh` must NOT override `CONJURE_HOME`.** Only `HOME`, `XDG_CONFIG_HOME`, `CLAUDE_CONFIG_DIR`, and `PATH` are sandboxed. (RESEARCH.md Pitfall 5)

5. **monorepo fixture seed must include `packages/api/` directory.** Without it, `profiles/monorepo/apply.sh` exits 1 without appending the fragment. (RESEARCH.md Critical Finding 6, Pitfall 4)
