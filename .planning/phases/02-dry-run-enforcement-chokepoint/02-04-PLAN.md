<read_first>

- .planning/phases/02-dry-run-enforcement-chokepoint/02-RESEARCH.md (compliance inventory — gdpr/soc2/pci: bare cat >> only)
</read_first>

**Action:** The three simple compliance overlays (gdpr, soc2, pci) each have a single
bare `cat "$PROFILE_DIR/CLAUDE.md.fragment" >> "$TARGET/CLAUDE.md"` with no DRY guard.
Apply a uniform 3-step migration:

**Step 1 — Add source line after PROFILE_DIR declaration:**
```
PROFILE_DIR="$(cd "$(dirname "$0")" && pwd)"
source "$CONJURE_HOME/lib/mutate.sh"   ← ADD this line
```
Unlike profiles, these scripts have NO `DRY=` line to remove — the header is cleaner.

**Step 2 — Replace bare append with mutate_write (Option B — inline command substitution):**
```
# Before (sentinel differs per overlay — gdpr/soc2/pci respectively):
cat "$PROFILE_DIR/CLAUDE.md.fragment" >> "$TARGET/CLAUDE.md"

# After:
mutate_write "$TARGET/CLAUDE.md" "$(cat "$PROFILE_DIR/CLAUDE.md.fragment")" "--append"
```

**Step 3 — Add mutate_summary before final echo:**
```
mutate_summary
echo "✓ <NAME> overlay applied"
```

Read each file before editing — confirm the exact sentinel comment and final echo text.
Apply the same pattern to all three. The surrounding `if [ -f "$TARGET/CLAUDE.md" ] && ! grep -q "<!-- compliance:X -->"` guard stays intact — only the bare write inside it changes.

## Task 2: Migrate hipaa (6 write operations including mkdir + cp + chmod)

**Type:** auto
**Files:** compliance/hipaa/apply.sh

<read_first>

- lib/mutate.sh (confirm mutate_mkdir, mutate_cp signatures from 02-01)
- .planning/phases/02-dry-run-enforcement-chokepoint/02-PATTERNS.md (compliance/hipaa section with all per-line replacements)
- .planning/phases/02-dry-run-enforcement-chokepoint/02-RESEARCH.md (hipaa inventory; chmod decision same as java-spring)
</read_first>

**Action:** HIPAA is the most complex overlay — 6 mutation operations, none currently guarded.
Read the current file first to see the exact line content before editing.

Add source line after `PROFILE_DIR` declaration (same as gdpr/soc2/pci above — no DRY to remove):
```
source "$CONJURE_HOME/lib/mutate.sh"
```

Replace all 6 bare write operations (per PATTERNS.md hipaa section):

1. Fragment append: `cat "$PROFILE_DIR/CLAUDE.md.fragment" >> "$TARGET/CLAUDE.md"` → `mutate_write "$TARGET/CLAUDE.md" "$(cat "$PROFILE_DIR/CLAUDE.md.fragment")" "--append"`

2. Hook dir mkdir: `mkdir -p "$TARGET/.claude/hooks"` → `mutate_mkdir "$TARGET/.claude/hooks"`

3. Hook cp: `cp "$PROFILE_DIR/pre-commit-phi-scan.sh" "$TARGET/.claude/hooks/" 2>/dev/null || true` → `mutate_cp "$PROFILE_DIR/pre-commit-phi-scan.sh" "$TARGET/.claude/hooks/pre-commit-phi-scan.sh"` (drop the `|| true` — if the source file is missing, the test suite would catch it; the explicit destination path is cleaner)

4. chmod: `chmod +x "$TARGET/.claude/hooks/pre-commit-phi-scan.sh" 2>/dev/null` → inline guard (D-02 excludes mutate_chmod): `[ "${DRY_RUN:-0}" = "1" ] || chmod +x "$TARGET/.claude/hooks/pre-commit-phi-scan.sh" 2>/dev/null`

5. Compliance docs mkdir: `mkdir -p "$TARGET/docs/compliance"` → `mutate_mkdir "$TARGET/docs/compliance"`

6. Controls cp: `cp "$PROFILE_DIR/CONTROLS.md" "$TARGET/docs/compliance/HIPAA-CONTROLS.md" 2>/dev/null || true` → `mutate_cp "$PROFILE_DIR/CONTROLS.md" "$TARGET/docs/compliance/HIPAA-CONTROLS.md"`

Add `mutate_summary` before the final `echo "✓ HIPAA overlay applied"` line.
Keep the `echo "  ⚠ Compliance ≠ Config. Engage your compliance officer."` line — it is informational output, not a mutation.
