**Change 1 — Line 75: Add DRY_RUN to init-project.sh invocation**

Template comes from the existing `cmd_migrate` pattern at line 107 (already proven working):
```bash
CONJURE_HOME="$CONJURE_HOME" DRY_RUN="$dryrun" bash "$script" "$target"
```

Apply the same prefix pattern:
```bash
# Before:
bash "$CONJURE_HOME/scripts/init-project.sh" "$mode" "$target"

# After:
CONJURE_HOME="$CONJURE_HOME" DRY_RUN="$dryrun" bash "$CONJURE_HOME/scripts/init-project.sh" "$mode" "$target"
```

**Change 2 — Line 80: Add DRY_RUN to profile invocation, remove positional $dryrun arg**

```bash
# Before:
bash "$CONJURE_HOME/profiles/$profile/apply.sh" "$target" "$dryrun"

# After (env var replaces positional arg — profiles now read DRY_RUN from env):
CONJURE_HOME="$CONJURE_HOME" DRY_RUN="$dryrun" bash "$CONJURE_HOME/profiles/$profile/apply.sh" "$target"
```

**Change 3 — Line 84: Replace bare echo redirect with mutate_write + mutate_summary**

Source `lib/mutate.sh` at the top of `cmd_init()` — add it immediately after the
`local` declarations at the function top (before the `while` argument-parsing loop),
so it is available for the version stamp at the bottom:
```bash
source "$CONJURE_HOME/lib/mutate.sh"
```

Then replace the bare write:
```bash
# Before:
echo "$CONJURE_VERSION" > "$target/.claude/.conjure-version"

# After:
mutate_write "$target/.claude/.conjure-version" "$CONJURE_VERSION"
mutate_summary
```

`mutate_summary` here accounts for the version stamp mutation only — the child scripts
(init-project.sh, apply.sh) each call their own `mutate_summary` for their own counter.
The CLI-level summary captures the inline stamp. This is correct per RESEARCH.md
Architectural Responsibility Map: "Per-script mutation counting/summary — Each script
that sources lib/mutate.sh".

Do NOT move the `echo "▸ Pinned conjure version: $CONJURE_VERSION"` line — it is
informational output, not a mutation, and should remain after mutate_write.

The file uses `set -uo pipefail` (line 22). The source line and all mutate_* calls
use `:-` defaults — safe under `set -u` (per Pitfall 4 / PATTERNS.md).
