**Action:** Insert a new "Dry-run enforcement (SAFE-01, SAFE-02)" section into
`tests/run.sh` immediately before the existing "Migration coverage" section. Follow
the exact `pass()`/`fail()` helper convention already in the file (lines 14-15).

The new section must implement these assertions (per RESEARCH.md Validation Architecture
and PATTERNS.md tests/run.sh section):

```bash
echo
echo "▸ Dry-run enforcement (SAFE-01, SAFE-02)"

TMPDIR_TARGET="$(mktemp -d)"
trap 'rm -rf "$TMPDIR_TARGET"' EXIT

# Create a minimal CLAUDE.md so profile/compliance fragments have something to append to
printf '# Test project\n' > "$TMPDIR_TARGET/CLAUDE.md"

# Run conjure init --dry-run against the temp dir
DRY_OUT="$(CONJURE_HOME="$CONJURE_HOME" cli/conjure init --dry-run "$TMPDIR_TARGET" 2>&1 || true)"

# SAFE-01 assertion: .claude/ must NOT be created
if [ -d "$TMPDIR_TARGET/.claude" ]; then
  fail "dry-run: .claude/ was created (filesystem mutated — SAFE-01)"
else
  pass "dry-run: .claude/ not created (SAFE-01)"
fi

# SAFE-01 / D-04 assertion: [dry-run] prefix lines must appear in output
if printf '%s' "$DRY_OUT" | grep -q "\[dry-run\]"; then
  pass "dry-run: [dry-run] prefix lines present in output (D-04)"
else
  fail "dry-run: no [dry-run] lines in output (D-04)"
fi

# D-05 assertion: mutation count > 0 in summary line
if printf '%s' "$DRY_OUT" | grep -qE "\[dry-run\] [1-9][0-9]* mutations skipped"; then
  pass "dry-run: mutation count > 0 in summary line (D-05)"
else
  fail "dry-run: summary line missing or count is 0 (D-05)"
fi
```

Placement: Insert immediately before the `echo` + `echo "▸ Migration coverage"` block.
Read the exact line number / surrounding context before inserting to avoid misplacement.

Notes on the test design:
- Use `|| true` on the conjure call so a non-zero exit does not abort the test suite under `set -uo pipefail`.
- The `CONJURE_HOME` variable is already set at the top of `tests/run.sh` — reuse it.
- The `TMPDIR_TARGET` trap uses `EXIT` — this is idiomatic in this test file (the preflight section already sets up similar environment cleanup).
- No `--profile` flag on the init call — this tests the base init path. Profile-specific dry-run is covered by the profile smoke tests added in 02-03.
- The `grep -qE "\[dry-run\] [1-9][0-9]* mutations skipped"` regex matches one or more digits (not zero) — confirming the count is meaningful.

Do NOT add a `CONJURE_VERSION` assertion on the target tree — the `.conjure-version` file test is covered implicitly by `[ ! -d "$TMPDIR_TARGET/.claude" ]` (no .claude/ = no version stamp possible).
