**Action:** Create `lib/mutate.sh` as a sourced bash library. Do NOT add `set -uo pipefail`
at the top — the library is sourced into callers that already have those options set; adding
them again is not the project idiom (per PATTERNS.md: "libraries do not set shell options").

The file must implement four functions (per D-01 / D-02 / D-05):

- `mutate_mkdir <dir>` — when DRY_RUN=1: print `[dry-run] would mkdir <dir>`, increment counter, return 0. When DRY_RUN=0/unset: run `mkdir -p "$1"`.

- `mutate_cp <src> <dest>` — when DRY_RUN=1: print `[dry-run] would cp <src> <dest>`, increment counter, return 0. When DRY_RUN=0/unset: detect `[ -d "$1" ]` and run `cp -r` or plain `cp` accordingly (per RESEARCH Pitfall 5 / PATTERNS.md mutate_cp block).

- `mutate_write <dest> <content> [--append]` — when DRY_RUN=1: print `[dry-run] would write <dest>`, increment counter, return 0. When DRY_RUN=0/unset: use `printf '%s\n' "$content"` redirected to dest (overwrite) or appended (--append flag as `$3`). Use `printf` not `echo` for portability (per PATTERNS.md "printf over echo"). Use `local` for `dest`, `content`, `mode` — the library may be sourced in contexts where `local` is valid (inside bash functions).

- `mutate_summary` — when DRY_RUN=1: print `[dry-run] ${CONJURE_DRY_MUTATION_COUNT} mutations skipped — run without --dry-run to apply`. When DRY_RUN=0/unset: no output, no-op.

Counter initialization at source time (idempotent, set -u safe):
`CONJURE_DRY_MUTATION_COUNT="${CONJURE_DRY_MUTATION_COUNT:-0}"`

Every variable access uses `${VAR:-default}` — never bare `$DRY_RUN` or `$CONJURE_DRY_MUTATION_COUNT` (per Pitfall 4 / PATTERNS.md "set -u safe variable access").

The `[dry-run]` prefix string must be exactly `[dry-run]` — no variants. Phase 4 golden-file tests will grep for this exact string (per D-04, CONTEXT.md specifics).

Do NOT add `mutate_mv`, `mutate_rm`, `mutate_chmod` — D-02 locks the minimal function set.
