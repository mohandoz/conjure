**Action:** Apply the identical 3-step migration to each of the 7 simple profiles.
Read each file before editing it — confirm the exact content, do not assume.

The 3-step migration per profile (per PATTERNS.md "profiles/ts-next/apply.sh" section):

**Step 1 — Remove DRY positional arg, add source line:**
```
# Remove this line (line 4 in all 7 profiles):
DRY="${2:-0}"

# Add this line after PROFILE_DIR= :
source "$CONJURE_HOME/lib/mutate.sh"
```

**Step 2 — Replace write guard with mutate_write (Option B — inline command substitution):**
```
# Before (in all 7 profiles, sentinel differs per profile):
[ "$DRY" = 0 ] && cat "$PROFILE_DIR/CLAUDE.md.fragment" >> "$TARGET/CLAUDE.md"

# After:
mutate_write "$TARGET/CLAUDE.md" "$(cat "$PROFILE_DIR/CLAUDE.md.fragment")" "--append"
```
Option B is correct: `$(cat ...)` runs in a subshell but its *return value* is captured;
`mutate_write` itself runs in the current shell so the counter increments correctly
(per RESEARCH.md Pitfall 3 — verified correct).

**Step 3 — Add mutate_summary before the final echo:**
```
mutate_summary
echo "✓ Profile <name> applied"
```

All 7 profiles have the exact same `cat "$PROFILE_DIR/CLAUDE.md.fragment" >> "$TARGET/CLAUDE.md"` pattern inside a `grep -q "<!-- profile:<name> -->"` guard. Apply uniformly.

**Action — java-spring:**

Apply the same 3-step header migration (remove `DRY="${2:-0}"`, add source line, add mutate_summary).

For the write operations (per PATTERNS.md java-spring section):
- Replace `[ "$DRY" = 0 ] && cat "$PROFILE_DIR/CLAUDE.md.fragment" >> "$TARGET/CLAUDE.md"` with `mutate_write "$TARGET/CLAUDE.md" "$(cat "$PROFILE_DIR/CLAUDE.md.fragment")" "--append"`
- Replace `[ "$DRY" = 0 ] && cp "$PROFILE_DIR/hooks/post-edit-format.sh" "$TARGET/.claude/hooks/post-edit-format.sh"` with `mutate_cp "$PROFILE_DIR/hooks/post-edit-format.sh" "$TARGET/.claude/hooks/post-edit-format.sh"`
- Replace `[ "$DRY" = 0 ] && chmod +x "$TARGET/.claude/hooks/post-edit-format.sh"` with the inline guard (D-02 excludes mutate_chmod): `[ "${DRY_RUN:-0}" = "1" ] || chmod +x "$TARGET/.claude/hooks/post-edit-format.sh"`

The chmod is naturally skipped in dry-run because mutate_cp exits early and the file
is never created — but the explicit inline guard is still correct for cases where the
file might pre-exist from a prior run.

**Action — monorepo:**

Apply the same 3-step header migration.

The monorepo heredoc write inside the package loop needs the variable-capture pattern
(Pitfall 3 — the `cat > "$pkg/CLAUDE.md" <<EOF` is currently inside `if [ "$DRY" = 0 ]`):

Read the EXACT content of the heredoc from the current file before writing the replacement.
Assign it to a variable with embedded newlines, then pass to mutate_write:

```bash
MONOREPO_CONTENT="# $dir/$name — Local Working Notes
<...exact content from current heredoc...>"
mutate_write "$pkg/CLAUDE.md" "$MONOREPO_CONTENT"
```

Any backtick characters in the heredoc content must be escaped as `\`` in the variable
string to prevent command substitution (per PATTERNS.md monorepo note).

Also replace the root CLAUDE.md append: `[ "$DRY" = 0 ] && cat "$PROFILE_DIR/CLAUDE.md.fragment" >> "$TARGET/CLAUDE.md"` with `mutate_write "$TARGET/CLAUDE.md" "$(cat "$PROFILE_DIR/CLAUDE.md.fragment")" "--append"`.
