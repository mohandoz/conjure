**Type:** auto

**Action:** Replace every bare write operation in `scripts/init-project.sh` with the
corresponding `mutate_*` function. Apply all changes in one edit pass (read the file,
build the complete replacement, write it). Do NOT do partial edits — the file should
be fully correct after this task.

Replacements to make (12 write sites, from the RESEARCH.md write-site inventory):

1. **Line 25** — `mkdir -p .claude/{skills,agents,hooks,docs}` → 4 explicit calls (Pitfall 2 — brace expansion cannot pass through a function):
   ```
   mutate_mkdir ".claude/skills"
   mutate_mkdir ".claude/agents"
   mutate_mkdir ".claude/hooks"
   mutate_mkdir ".claude/docs"
   ```

2. **Lines 29-35 loop** — `cp "$KIT/templates/$f" "$f"` → `mutate_cp "$KIT/templates/$f" "$f"`

3. **Line 39** — `cp "$KIT/templates/settings.json.tmpl" .claude/settings.json` → `mutate_cp "$KIT/templates/settings.json.tmpl" ".claude/settings.json"`

4. **Line 49** — `cp "$hook" ".claude/hooks/$name"` → `mutate_cp "$hook" ".claude/hooks/$name"`

5. **Line 57** — `cp -r "$KIT/templates/skills/$skill" ".claude/skills/$skill"` → `mutate_cp "$KIT/templates/skills/$skill" ".claude/skills/$skill"` (mutate_cp auto-detects -r via `[ -d "$1" ]`)

6. **Line 65** — `cp -r "$KIT/templates/skills/$skill" ".claude/skills/$skill"` → `mutate_cp "$KIT/templates/skills/$skill" ".claude/skills/$skill"`

7. **Line 73** — `cp "$KIT/templates/agents/$agent" ".claude/agents/$agent"` → `mutate_cp "$KIT/templates/agents/$agent" ".claude/agents/$agent"`

8. **Line 79** — `mkdir -p docs/adr` → `mutate_mkdir "docs/adr"`

9. **Lines 81-84 loop** — `cp "$KIT/templates/docs/$doc.md.tmpl" "docs/$doc.md"` → `mutate_cp "$KIT/templates/docs/$doc.md.tmpl" "docs/$doc.md"`

10. **Line 87-89** — `cp "$KIT/templates/docs/ADR-TEMPLATE.md" docs/adr/0001-*.md` → `mutate_cp "$KIT/templates/docs/ADR-TEMPLATE.md" "docs/adr/0001-record-architecture-decisions.md"`

11. **Lines 93-109** — heredoc `cat >.env.example <<'EOF' ... EOF` → variable capture pattern (Pitfall 3 — pipe creates subshell and loses counter):
    ```bash
    if [ ! -f .env.example ]; then
      ENV_CONTENT='# .env.example — every env var, with placeholder values.
    # Real .env is gitignored.
    #
    # Add each new env var here when the code references one.

    # Database
    # DATABASE_URL=postgresql://user:pass@localhost:5432/dbname

    # External services
    # REASONER_BASE_URL=http://localhost:9009

    # Secrets (placeholder values only)
    # API_KEY=changeme'
      mutate_write ".env.example" "$ENV_CONTENT"
      echo "  ✓ created .env.example"
    fi
    ```
    Read the ACTUAL heredoc content from the file before writing the replacement — use
    the exact content verbatim to avoid any unintended change.

12. **Line 113** — `[ -f .claude/COMPOUND-CANDIDATES.md ] || echo "..." > .claude/COMPOUND-CANDIDATES.md` → expand to explicit if block:
    ```bash
    if [ ! -f .claude/COMPOUND-CANDIDATES.md ]; then
      mutate_write ".claude/COMPOUND-CANDIDATES.md" "# Compound Engineering — Candidate Rules from Sessions"
    fi
    ```

After all replacements, add `mutate_summary` call at the tail of the script, immediately
before the `cat <<EOF` next-steps banner (line ~116). This emits the count line only when
DRY_RUN=1 (the function is a no-op in live mode).

Verify zero bare write operations remain after edit: no `mkdir `, no `cp `, no `cat >`,
no `echo.*>` outside of comments. The only filesystem operations should be via `mutate_*`
functions.
