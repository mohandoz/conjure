<objective>
Ship the skill-telemetry hook, schema documentation, settings wiring, and gitignore guard — the complete foundation layer for opt-in, local-only, PII-free skill telemetry.

Purpose: Users who set CONJURE_TELEMETRY=1 in their project's .claude/settings.json env block will have every skill invocation appended as a JSONL event to .claude/telemetry/skill-events.jsonl. Users who do not opt in (or who have DO_NOT_TRACK=1) get zero writes and zero overhead. The schema documentation makes the no-egress guarantee verifiable by any user. This is Wave 1 — the audit integration and tests in later waves depend on the hook design and schema being finalized here.

Output: templates/hooks-nodejs/skill-telemetry.mjs, TELEMETRY.md, updated templates/settings.json.tmpl, updated templates/.gitignore.tmpl

    - When CONJURE_TELEMETRY !== '1': process.exit(0) immediately — no log written, per D-01/D-02
    - When stdin JSON parse fails (malformed or empty): process.exit(0) — hook must never crash
    - When hook_event_name is PreToolUse and tool_name is Skill: writes record with event: 'skill_invoke', skill: p.tool_input?.skill_name
    - When hook_event_name is UserPromptExpansion: writes record with event: 'skill_typed', skill: p.command_name (strip leading '/' defensively)
    - When hook event is neither PreToolUse/Skill nor UserPromptExpansion: process.exit(0) — silent pass
    - When skillName is null (field missing from payload): process.exit(0) — silent pass
    - JSONL record fields: ts (ISO 8601 UTC), session_id, event, skill (name only), project_cwd — never skill_args or command_args
    - Log directory .claude/telemetry/ created with mkdirSync({recursive:true}) on first write — idempotent
    - fs errors during write are caught and silently ignored — telemetry must never block Claude
    - stdout is never written to — all output goes to the log file or is silently dropped
    - 5-second stdin guard: setTimeout(() => process.exit(0), 5000) before stdin listener, cleared in 'end' handler
    - All hook logic lives inside process.stdin.on('end', callback) — never call process.exit() outside it
  </behavior>
  <action>
    Create templates/hooks-nodejs/skill-telemetry.mjs as an ES module (.mjs) with shebang #!/usr/bin/env node.

    File structure (in order):
    1. Shebang line + comment block (per D-03, D-04): explain it handles PreToolUse(Skill) + UserPromptExpansion, opt-in via CONJURE_TELEMETRY=1, exit 0 always
    2. Imports: import { mkdirSync, appendFileSync } from 'node:fs' and import path from 'node:path' — no other imports
    3. DO_NOT_TRACK check FIRST (per D-02, Unix convention): if (process.env.DO_NOT_TRACK === '1') process.exit(0)
    4. CONJURE_TELEMETRY check SECOND (per D-01): if (process.env.CONJURE_TELEMETRY !== '1') process.exit(0)
    5. 5-second stdin guard: const guard = setTimeout(() => process.exit(0), 5000)
    6. Stdin accumulator: let raw = ''; process.stdin.setEncoding('utf8'); process.stdin.on('data', chunk => { raw += chunk; })
    7. On 'end' callback — ALL remaining logic inside:
       a. clearTimeout(guard)
       b. JSON.parse(raw) inside try/catch — exit 0 on parse failure
       c. Branch on p.hook_event_name:
          - 'PreToolUse' + p.tool_name === 'Skill': skillName = p.tool_input?.skill_name ?? null, eventType = 'skill_invoke'
          - 'UserPromptExpansion': skillName = p.command_name ?? null; if (skillName) skillName = skillName.replace(/^\//, ''); eventType = 'skill_typed'
          - else: process.exit(0)
       d. if (!skillName) process.exit(0) — defensive null guard (per A1/A2 assumptions)
       e. Build record: JSON.stringify({ ts: new Date().toISOString(), session_id: p.session_id, event: eventType, skill: skillName, project_cwd: p.cwd })
       f. try { const logDir = path.join(p.cwd, '.claude', 'telemetry'); mkdirSync(logDir, { recursive: true }); appendFileSync(path.join(logDir, 'skill-events.jsonl'), record + '\n'); } catch { /* silent */ }
       g. process.exit(0)

    CRITICAL anti-patterns to avoid (per RESEARCH.md):
    - Never write to stdout (CC parses stdout at exit 0 as JSON context — any output breaks the session)
    - Never exit 2 (blocks Claude from using the skill)
    - Never log skill_args or command_args (PII risk, D-05)
    - Never call process.exit() outside the 'end' callback (except for the early-exit guards at lines 3–4)
    - Never use string concat with '/' for paths — use path.join

    After creating the file, run: chmod +x templates/hooks-nodejs/skill-telemetry.mjs
