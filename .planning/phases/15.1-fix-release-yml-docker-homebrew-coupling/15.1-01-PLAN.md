# Plan 15.1-01: Decouple Docker and Homebrew jobs in release.yml

**Phase:** 15.1 — Fix release.yml Docker+Homebrew coupling
**Status:** completed
**Requirements:** REL-01 (tech debt), DOCK-03 (tech debt)

## Problem

`release` job had Docker push and Homebrew bump as sequential steps in the same job.
If Docker push failed, Homebrew bump was silently skipped with no independent failure signal.
No preflight check for HOMEBREW_TAP_GITHUB_TOKEN before the bump step.

## Solution

Split monolithic `release` job into 4 independent jobs:

```
ci-gate
  ↓
release (VERSION check + marketplace check + CHANGELOG + GH release create)
  ↓ (both need: release, run in parallel)
docker                          homebrew
(setup + build + push)          (token preflight + formula bump)
```

## Changes

**File:** `.github/workflows/release.yml`

- `release` job: retains `contents: write` only (drops `packages: write`)
  - Steps: checkout, VERSION check, marketplace check, CHANGELOG extract, GH release create
- `docker` job (new, `needs: [release]`): `packages: write`
  - Steps: checkout, containerd setup, QEMU, Buildx, GHCR login, build+push
- `homebrew` job (new, `needs: [release]`): no special permissions
  - Step 1: preflight — exits 0 with warning if `COMMITTER_TOKEN` env is empty
  - Step 2: bump action — gated with `if: ${{ env.COMMITTER_TOKEN != '' }}`

## Verification

- Docker failure now surfaces as `docker` job failure without blocking `homebrew` job
- Missing HOMEBREW_TAP_GITHUB_TOKEN prints clear warning, doesn't fail the run
- CHANGELOG extraction no longer blocked by Docker setup steps
