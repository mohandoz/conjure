Module-level state (after header, before functions):

- CONJURE_MERGE_CONFLICT_FILES=""

Function 1 — merge_file_3way:
Signature: merge_file_3way() with locals: current="$1" base="$2" new="$3" rel="$4" pinned_ver="$5" new_ver="$6"

Body logic (per D-03, D-05, RESEARCH.md verified behavior):
1. Capture stdout+rc from: git merge-file -p --diff3 -L "your version ($rel)" -L "v${pinned_ver} base" -L "v${new_ver} upstream" "$current" "$base" "$new" 2>/dev/null
   Store in locals merged and rc (separate lines: merged=... then rc=$?)
2. Three-way branch on rc:
   - rc == 0: call mutate_write "$current" "$merged"; return 0
   - rc > 0 && rc < 255: call write_merge_sidecar "$current" "$rel" "$merged"; return 1
   - rc == 255: print "✗ git merge-file error on ${rel} (rc=${rc})" to stderr; return 2
3. CRITICAL: Never use in-place mode (no -p omission). Never check rc with plain [ $rc -ne 0 ] — must distinguish conflict from error.

Function 2 — write_merge_sidecar:
Signature: write_merge_sidecar() with locals: current_file="$1" rel="$2" content="$3"

Body logic (per D-04):
1. DRY_RUN guard first: if [ "${DRY_RUN:-0}" = "1" ]; then echo "[dry-run] would write sidecar for ${rel}"; CONJURE_DRY_MUTATION_COUNT=$((CONJURE_DRY_MUTATION_COUNT+1)); return 0; fi
2. Encode: local encoded; encoded="$(printf '%s' "$rel" | tr '/' '_')"
3. Sidecar name: local sidecar_name=".conjure-conflict-${encoded}"
4. Sidecar dir: local sidecar_dir; sidecar_dir="$(dirname "$current_file")"
5. Sidecar path: local sidecar_path="${sidecar_dir}/${sidecar_name}"
6. Write via: printf '%s\n' "$content" > "$sidecar_path"
7. Track: CONJURE_MERGE_CONFLICT_COUNT=$((CONJURE_MERGE_CONFLICT_COUNT+1)); CONJURE_MERGE_CONFLICT_FILES="${CONJURE_MERGE_CONFLICT_FILES} ${sidecar_path}"
Note: write_merge_sidecar calls printf directly (not mutate_write) because it already checks DRY_RUN itself and must write the sidecar (not the current file). The DRY_RUN guard at the top of the function covers the dry-run case.

Function 3 — merge_user_files:
Signature: merge_user_files() with locals: target="$1" snap_dir="$2" conjure_ver="$3" pinned_ver="$4"

Body logic (per RESEARCH.md architecture diagram + PATTERNS.md file iteration pattern):
This function iterates all user-owned template types and calls merge_file_3way for each existing installed file.

Reset conflict tracking at start: CONJURE_MERGE_CONFLICT_COUNT=0; CONJURE_MERGE_CONFLICT_FILES=""

CLAUDE.md template (single file):
- tmpl_file="$CONJURE_HOME/templates/CLAUDE.md.tmpl"
- current="$target/.claude/CLAUDE.md" (NOTE: installed as CLAUDE.md not CLAUDE.md.tmpl)
- base="$snap_dir/CLAUDE.md.tmpl"
- rel="CLAUDE.md"
- If both current and base exist: call merge_file_3way "$current" "$base" "$tmpl_file" "$rel" "$pinned_ver" "$conjure_ver"; handle rc

Skills (find loop, POSIX-safe with while IFS= read -r):
- Find: find "$CONJURE_HOME/templates/skills" -name SKILL.md
- For each tmpl_file: derive rel="${tmpl_file#$CONJURE_HOME/templates/}" (gives "skills/name/SKILL.md")

Agents (same pattern, find "$CONJURE_HOME/templates/agents" -name '*.md'):
