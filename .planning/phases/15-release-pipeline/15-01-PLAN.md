<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub API → ci-gate | gh api response is parsed with jq; malformed JSON would skip failure detection |
| GITHUB_TOKEN → ghcr.io push | Built-in token is scoped to the repository; packages: write must be explicitly granted |
| HOMEBREW_TAP_GITHUB_TOKEN → external tap repo | Secret must be set in repo settings; workflow fails if absent |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Tampering | ci-gate gh api check-runs | mitigate | Filter by check name != "Release" to prevent self-referential bypass; jq select on conclusion field |
| T-15-02 | Elevation of Privilege | packages: write on release job | mitigate | Permission is job-scoped (not workflow-scoped); ci-gate has no write permissions |
| T-15-03 | Spoofing | ghcr.io login via GITHUB_TOKEN | accept | GITHUB_TOKEN is cryptographically issued per workflow run by GitHub; no long-lived credential |
| T-15-04 | Denial of Service | docker/build-push-action timeout on multi-arch | accept | GitHub Actions has 6-hour job timeout; multi-arch builds for this image size are well within limit |
| T-15-SC | Tampering | No new npm/pip/cargo installs in this phase | accept | No package manager installs; all actions are pinned to explicit major versions matching docker.yml |
</threat_model>
