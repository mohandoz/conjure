## Overview

v0.3.0 closes the gap between what Conjure *claims* and what it *verifiably does*.
The journey was dependency-ordered: first fix the two live bugs that undermine
trust (`--dry-run` mutates disk; Windows hook wiring is silently dead) and harden
pre-flight, then build sandboxed per-profile fixtures, then the regression net
that guards everything — proving dry-run is byte-safe. Once `init` was trustworthy
and verifiable, we recorded the README demo, shipped the offline cost estimator,
and landed the headline differentiator (local-only skill-firing telemetry) last,
with the most verification.

## Milestone Summary

### Key Decisions

- All writes funnel through `lib/mutate.sh` chokepoint — dry-run enforced once, not per call site
- `node .mjs` hook wiring universally in settings template with relative paths — no OS branching, no shell arg expansion
- Telemetry: local-only, opt-in, PII-free by design; no-egress is an enforced CI test, not a promise
- chars/4 heuristic for cost estimation — no bundled tokenizer (none accurate exists), `--exact` opt-in for API-backed count

### Issues Resolved

- Live bug: `--dry-run` was mutating disk (init-project.sh, profiles, version stamp) — fixed via SAFE-01/02
- Live bug: bash hooks silently dead on native Windows — fixed via SAFE-03 (node .mjs universal)
- Duplicate preflight logic inlined across CLI and scripts — extracted to `scripts/preflight.sh` (SAFE-04)

### Issues Deferred

- `cmd_update --apply` 3-way merge — placeholder stub since v0.2.x; documented in FAILURE-MODES.md; v0.4.0 scope
- Windows CI runtime for TEST-06 — code correct, needs push to GitHub to run on `windows-latest`
- GIF animation content visual review (DOCS-01) — needs human to open in viewer and check GitHub rendering
